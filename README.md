## Обзор

**Typr** — это легковесное приложение для голосового ввода (speech-to-text), работающее из системного трея. Зажмите горячую клавишу для записи голоса, отпустите её для транскрипции — и распознанный текст автоматически вставится в любое активное текстовое поле. Поддерживается работа как с официальным OpenAI Whisper API, так и с любым совместимым локальным эндпоинтом.

**Ключевые особенности:**
- 🎤 **Push-to-talk** — удерживайте хоткей для записи, отпустите для транскрипции и вставки.
- ⌨️ **Вставка в любом месте** — текст вставляется в активное окно любого приложения.
- 🌐 **Независимость от раскладки** — корректный Unicode-ввод через clipboard backend.
- 🐧 **Linux-first** — поддержка Wayland и X11 через `evdev`/`uinput`.
- 🔌 **Hotplug** — автообнаружение новых клавиатур во время работы.
- 🏠 **Локальные endpoint'ы** — совместимо с OpenAI API и локальными STT-серверами (включая `parakeet-tdt`).

## Возможности

### Основной функционал
- **Запись по нажатию (Push-to-Talk)** — удерживайте клавишу для записи, отпустите для завершения и вставки.
- **Универсальная вставка текста** — мгновенная раскладочно-независимая вставка текста с помощью виртуального устройства ввода уровня ядра.
- **Защита буфера обмена** — перед вставкой текущий текст из вашего буфера обмена сохраняется и автоматически восстанавливается через 3 секунды. Если вы скопируете что-то новое во время ожидания, восстановление отменится, чтобы не перезаписать ваши свежие данные.
- **Интеграция с треем** — работа в фоновом режиме с цветовыми индикаторами статуса приложения (запись, отправка, ошибка).
- **Запись кликом мыши** — альтернативный режим: левый клик по иконке в трее для старта записи, повторный клик для остановки.

### Гибкость настройки
- **Пользовательские API эндпоинты** — используйте OpenAI, локальные Whisper-сервера или локальный контейнер **parakeet-tdt**.
- **Динамический поиск моделей** — кнопка «Fetch» в настройках запрашивает список доступных моделей с вашего эндпоинта.
- **Настраиваемые горячие клавиши** — возможность задать любое удобное сочетание клавиш для записи.
- **История транскрипций** — сохранение истории записей локально с удобным поиском, копированием и удалением записей.

### Технические детали
- **Поддержка Wayland и X11** — работа через подсистему `evdev` не зависит от используемого композитора.
- **Автоматическая настройка прав** — установщик сам создает правила `udev` для `/dev/uinput` и добавляет пользователя в группу `input` для работы без root-прав.
- **Улучшенная отзывчивость** — сетевой тайм-аут Whisper сокращен до 15 секунд для предотвращения зависания статуса.
- **Адаптивный установщик** — выбирает только нужный clipboard package (`wl-clipboard` для Wayland, `xclip`/`xsel` для X11), а не устанавливает все сразу.
- **Preflight-проверка зависимостей** — перед установкой показывает полный список отсутствующих пакетов и запрашивает одно подтверждение.
- **Локализация установщика** — сообщения в `install.sh` на русском при `ru`-локали консоли, иначе на английском.

## Совместимость

Typr протестирован и полноценно поддерживается на следующих платформах:

*   **Дистрибутивы:**
    *   **Arch-based** (Arch Linux, Manjaro, CachyOS, EndeavourOS и др.)
    *   **Debian-based** (Ubuntu, Debian, Linux Mint, Pop!_OS, Zorin OS и др.)
    *   **Fedora-based** (Fedora Workstation, Silverblue и др.)
*   **Окружения рабочего стола (Desktop Environments):**
    *   **KDE Plasma, XFCE, Cinnamon, MATE, LXQt, LXDE, Budgie** — работают из коробки, значок приложения отображается в системном трее.
    *   **GNOME** (окружение по умолчанию в Ubuntu/Fedora) — поддерживается, но требует установки расширения GNOME Shell [AppIndicator and KStatusNotifierItem Support](https://extensions.gnome.org/extension/615/appindicator-support/) для отображения иконки настроек в трее.
    *   **Оконные менеджеры** (Sway, Hyprland, i3, AwesomeWM, Openbox и др.) — поддерживаются, но требуют наличия запущенного статус-бара с системным треем (например, `waybar`, `polybar`, `tint2`).

## Установка

### Быстрая установка

Запустите универсальный установщик. Он:
- определит дистрибутив, DE/WM и тип сессии (Wayland/X11);
- выполнит полную preflight-проверку зависимостей;
- покажет список отсутствующих пакетов и попросит одно подтверждение;
- запросит `sudo` только перед реальной установкой;
- предложит опционально развернуть локальный STT-контейнер Parakeet;
- настроит Python venv, desktop entries и `udev`-правила.

```bash
# Клонируйте репозиторий
git clone https://github.com/murtaza-nasir/typr.git
cd typr

# Запустите интерактивный установщик
./install.sh
```

Проверка без изменений в системе:
```bash
./install.sh --dry-run
```

`--check` работает как алиас `--dry-run`.

### Опциональный локальный STT (Parakeet)

В начале установки скрипт спрашивает, нужно ли локальное распознавание речи в Docker.

Если ответ `yes`, установщик:
- использует каталог `~/docker/parakeet-tdt`;
- если каталога нет, автоматически клонирует репозиторий  
  `https://github.com/groxaxo/parakeet-tdt-0.6b-v3-fastapi-openai`;
- запускает контейнер через `docker compose up -d` (или `docker-compose up -d`);
- при необходимости запрашивает запуск через `sudo`.

После успешного запуска выводится endpoint для настроек Typr:
`http://127.0.0.1:5092/v1`

> [!NOTE]
> После первой установки может потребоваться выйти из системы (Log Out) и зайти снова, чтобы применились права группы `input`.

### Проверка настройки

Проверьте, что ваш пользователь состоит в группе `input`:
```bash
groups | grep input
```

Запустите Typr:
```bash
# Из папки проекта:
./.venv/bin/typr
```

## Использование

1. **Запуск** — запустите Typr из меню ваших приложений или через терминал с помощью команды `./.venv/bin/typr` (находясь в папке проекта).
2. **Настройка** — нажмите правой кнопкой мыши по иконке в трее → **Settings**.
   - Перейдите во вкладку **API** → введите `Base URL` и API-ключ (если требуется).
   - Для локального Parakeet используйте `Base URL`: `http://127.0.0.1:5092/v1`.
   - Во вкладке **General** → нажмите **Fetch**, чтобы получить список доступных моделей с сервера, и выберите нужную.
3. **Запись** — **зажмите** сочетание клавиш `Meta+Shift+Space` (по умолчанию) и говорите.
4. **Вставка** — **отпустите** клавиши. Текст распознается и вставится в текстовое поле, где стоял фокус.

## Настройка

Конфигурационный файл хранится по пути: `~/.config/typr/config.json`

### Горячие клавиши

Настройка доступна через графический интерфейс (Settings → Hotkeys) или вручную в конфигурационном файле:

```json
{
  "hotkeys": {
    "push_to_talk": "Meta+Shift+Space"
  }
}
```

Поддерживаемый формат: `Модификатор+Модификатор+Клавиша` (например, `Meta+Shift+Space`).

## Устранение неполадок

### Ошибка «No keyboard devices found» или проблемы со вставкой текста
Убедитесь, что ваш пользователь добавлен в группу `input`:
```bash
groups | grep input
```
Если группы нет, выполните:
```bash
sudo usermod -aG input $USER
```
После этого обязательно выйдите из системы и зайдите снова (или перезагрузите ПК).

### Ошибки прав доступа к `/dev/uinput`
Убедитесь, что файл `/etc/udev/rules.d/99-uinput.rules` существует и содержит строку:
```udev
KERNEL=="uinput", GROUP="input", MODE="0660"
```
Установщик `install.sh` делает это автоматически, но в некоторых системах может потребоваться ручная перезагрузка правил:
```bash
sudo udevadm control --reload-rules && sudo udevadm trigger
```

### Зависание статуса «Transcribing...»
Тайм-аут ожидания сети составляет 15 секунд. Если по истечении этого времени транскрипция не завершилась, проверьте доступность вашего API-сервера с помощью кнопки **Test Connection** во вкладке **API** окна настроек.

## Удаление приложения

Для полной очистки системы от интеграции Typr и удаления виртуального окружения запустите деинсталлятор из папки проекта:

```bash
./uninstall.sh
```
`uninstall.sh` удаляет:
- desktop/autostart интеграцию Typr;
- локальные настройки, кэш, историю и `.venv`;
- legacy user-level `pip` install (если был).

Дополнительно скрипт интерактивно предлагает:
- удалить локальный Parakeet STT Docker-контейнер (`docker compose down` в `~/docker/parakeet-tdt`);
- удалить каталог `~/docker/parakeet-tdt` целиком (включая модели).

Примечание: папка самого репозитория Typr не удаляется.

## Стек технологий

- **Язык программирования**: Python 3.10+
- **Интерфейс (GUI)**: PyQt6
- **Аудио**: PyAudio (совместим с PipeWire и PulseAudio)
- **Низкоуровневый ввод**: evdev / uinput (прямой доступ к устройствам ввода Linux)
- **Сетевой клиент**: httpx
- **Clipboard backends**: `wl-clipboard` (Wayland), `xclip`/`xsel` (X11; выбирается автоматически установщиком)
